# 1.0.5（2026-02-10）

## 主要更新
- **category_name/category_type 列表化**：2min 输出 schema 中 `category_type`、`category_name`、`subtask_name` 从单值字符串改为列表（1~3 个元素），支持一个 2min 时段内多活动归类。
- **移除 task_label**：`task_label` 字段从 2min schema 中完全移除，由 `category_name` 列表取代其跨粒度聚合主键作用。
- **10min/1h prompt 适配**：10min prompt 的活动时间轴构建规则更新为从 `category_name` 列表展开各活动；1h prompt 保持不变（从 10min 的 activity_timeline 聚合）。
- **旧数据兼容策略**：无 `category_type` 的旧数据默认视为"行为"类型；无 `category_name` 但有 `task_label` 的旧数据用 `task_label[0]` 作为 label。旧数据不会被录入 Todo 模块行为目录。
- **no-change 模板适配**：2min 级无变化模板记录中 `task_label` 替换为 `category_type`/`category_name`/`subtask_name` 列表格式。
- **TodoWriter 适配**：回写逻辑支持新的列表格式（逐项处理），同时兼容旧的字符串格式。
- **前端 2min 展示适配**：2min 总结卡片中"任务标签"替换为"活动归类"，显示 `[类型] 名称` 格式，兼容旧数据回退到 task_label。
- **前端时间线兼容**：时间线泳道和筛选器中无 `category_type` 的旧数据默认显示为"行为"类型。

## 影响范围
- AI 总结：`ai_summary/src/prompt-builder.js`（2min/10min/1h prompt schema 和注入规则）、`ai_summary/src/screenshot-comparer.js`（no-change 模板）、`ai_summary/src/todo-writer.js`（列表格式处理）
- 前端：`electron-app/renderer/app.js`（2min 总结展示、时间线数据构建、筛选器默认值）
- 文档：`llmdoc/architecture/prompt-schema-design.md`、`llmdoc/architecture/ai-summary-architecture.md`
- 版本：`electron-app/package.json`（`1.0.4` -> `1.0.5`）
